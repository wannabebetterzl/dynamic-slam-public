[[STSLAM复刻]]的直接复现验证不可行，考虑从其它方面入手。

目标对象： **stereo / RGB-D**，非Mono单目

已开源代码借鉴：
<mark style="background:#fff88f"><font color="#000000">**1.VDO-SLAM（不够新，所以尚未实际应用）：**</font></mark>  

[VDO-SLAM](https://github.com/halajun/VDO_SLAM)是非常值得借鉴的。它是RGB-D 动态对象 SLAM，明确估计：
- camera pose
- static structure
- dynamic structure
- 每个刚体对象的 SE(3) motion
- object velocity

它的思想比 STSLAM 更工程可落地：不依赖复杂 shape factor，而是把动态对象作为 SE3 运动实体建模。对我们来说，最值得借鉴的是：

`普通静态点和动态对象点分离 动态对象有自己的 SE3 motion 相机位姿、静态点、动态点、物体运动统一优化`
**2. DynoSAM**
[DynoSAM](https://github.com/ACFR-RPG/DynOSAM) 是目前最值得重点参考的开源动态 SLAM 后端框架。它明确是一个 smoothing and mapping framework，用 factor graph 同时估计：

- camera poses
- static scene
- object motion / object poses
- object structures

它的价值在于：它不是只给一个系统结果，而是把动态 SLAM 的不同优化 formulation 做成可比较框架。对我们当前困境很有用，因为我们现在的问题正是“不知道动态后端到底应该怎么稳定进入图”。

**3. ClusterSLAM**

[ClusterSLAM](https://huangjh-pub.github.io/publication/clusterslam/) 不是我优先建议直接移植的工程代码方向，但它的后端思想很重要：

`不要完全相信语义 instance id 根据 3D landmarks 的运动一致性，把点聚成刚体 cluster 再估计每个 cluster 的运动`

这对我们很关键。我们现在过度依赖前端 instance id，一旦 ID、mask、深度、ORB tracklet 任一环不稳，后端就崩。ClusterSLAM 的启发是：**动态实体应该由几何运动一致性确认，而不是由语义 mask 直接决定。**

**4. MID-Fusion / Co-Fusion / MaskFusion**（后续高斯溅射或密集三维重建可考虑）

[MID-Fusion](https://github.com/ethz-mrl/mid-fusion)、[Co-Fusion](https://arxiv.org/abs/1706.06629)、[MaskFusion](https://github.com/martinruenz/maskfusion) 更偏 dense RGB-D object-level SLAM。

它们的共同思想是：

`相机跟踪尽量依赖静态背景 每个动态物体有独立模型 每个物体单独 tracking / fusion / pose estimation`

这类方法不一定适合直接塞进 ORB-SLAM3，但很适合借鉴架构边界：**动态对象是独立模型，不应该先进入普通静态地图再被后端“纠正”。**


CLusterSLAM是针对后端的，专注于从**运动一致性**角度为前端提供“动静分离”信息，其独立轨迹精度优化能力相对有限。优化核心为聚类，而非轨迹，为SLAM前端提供更“干净”和“结构化”的数据，而非直接给出最终定位结果。

## 研究尝试

先尝试将basic model和Cluster SLAM结合起来，作为前端和后端测试，ClusterSLAM未开源，暂时放弃

**现在尝试dynosam与basic model的结合：**
前端过滤动态物体后的信息参与静态地图的建立；
动态物体上的点、mask、深度、track id 会作为 object measurements，进入对象位姿、对象运动、对象结构估计；
后端因子图同时估计 camera pose、static map、object pose/motion、object structure，估计相机里程计、静态地图、物体运动/位姿和物体结构，把静态和动态测量放进统一优化问题中。

**Dynosam**

DynoSAM 的一大创新点，正是**不需要依赖语义识别或分割模型，仅通过纯几何和运动信息，就能估计出场景中物体的运动与姿态**。其核心在于将“检测与分割”过程，巧妙地**内化**在了一个联合优化的后端框架中。

这种方式建立在一个集成框架之上，该框架将静态与动态测量值统一到一个因子图优化问题中，从而同步求解包括物体运动和姿态在内的所有变量。

下面，我们来拆解下它的具体工作流程：

### 👣 第一步：前端数据关联与初步“动静”分离

DynoSAM 的前端依赖传统的视觉里程计（VO，如 ORB-SLAM3）来提供相机位姿的初始估计和特征点追踪。此时，它**并不区分**哪些是静态背景点，哪些是动态物体上的点。

- **初始化**：系统启动时，会暂时**假设整个场景是静态的**。
    
- **识别“异常”**：当场景中的物体发生运动时，它们上面的特征点对传统的静态VO模型会产生巨大的、系统性的重投影误差被传统SLAM当作“异常值”（outliers）而剔除[](https://arxiv.org/html/2501.11893v3)。
    
- **收集“动态”线索**：DynoSAM 非但不会丢弃这些被剔除的点，反而将它们视为宝贵的动态线索。系统会将这些特征点收集起来，为后端因子图提供“动态”测量值[](https://arxiv.org/html/2501.11893v3)。
    

### 🧠 第二步：后端因子图的联合优化与“聚类”

这是 DynoSAM 实现“去语义化”感知的核心。它将所有从静态和动态点收集到的信息，都放入一个统一的优化问题中。

1. **统一的优化问题**：将相机位姿、静态地图点、以及**多个假设存在的运动物体的轨迹和结构**，全部作为待求解的变量。
    
2. **数据驱动**：特征点会因为各自的“运动一致性”而自然形成簇。例如，一个物体上的点倾向于保持彼此间的相对几何关系不变（即刚体约束），并以相同的方式运动。
    
3. **自发性聚类**：系统通过最小化全局误差，**自发地将那些共享相同基础运动的特征点“聚类”到一起，并与共享不同运动的点分离开**。这个聚类过程是优化过程的自然结果，不需要任何外部的语义输入或运动分割模型。
    
4. **运动、姿态与结构的同步求解**：一旦特征点被划分为不同的运动子集，系统就能同步计算出每个部分的运动和姿态。论文为此提出了一种新颖的表述，将**刚体运动模型编码到物体姿态估计中**，使优化出的结果物理上更合理[](https://arxiv.org/html/2501.11893v3)。
 通过这种方式，DynoSAM实现了在没有语义数据时为动态物体估计精确的运动模型，这对于自动驾驶等**数据稀缺或物体种类未知（如太空探索）** 的场景尤为关键。

###  精确运动模型的工程与学术价值

DynoSAM后端在因子图框架中联合优化所得的**高精度、带ID、有时序的3D物体轨迹**，构成了时空场景的数字化骨架。下表对比了拥有与不拥有这一核心能力，系统在关键下游任务中的能力表现差异：

|工程应用场景|❌ 无精确运动模型（仅有静态地图）|✅ 拥有精确运动模型|
|---|---|---|
|**移动机器人导航与避障**|无法区分静止障碍物与移动障碍物，容易误入运动路径，且无法预测障碍物未来位置|可精确预测周围物体下一时刻位置，实现动态避障；在复杂施工场地等环境中，使机器人能够提前规划出一条**安全、平滑且高效的路径**|
|**自动驾驶感知与预测**|作为运动目标检测的附属品，通常只提供检测框，缺乏精确的3D轨迹和速度估计|行为预测：基于精确历史3D轨迹，可判断车辆将**左转、直行还是并线**；安全冗余：在恶劣天气等传感器退化场景，高质量的3D运动先验可作为感知模块的备份|
|**AR/MR虚实交互**|虚拟内容无法与真实动态物体对齐，AR体验割裂|实现"虚实互动"：虚拟球能与真实滚动的球发生精准碰撞，虚拟宠物可跳上移动的扫地机器人并随之移动，完成物理上可信的交互|
|**视频分析与安防监控**|在稠密人群场景易发生身份错乱（ID Switch），难以持续追踪特定目标|凭借时序一致性，在复杂场景做到**稳健的身份保持**；支持跨相机接力追踪，实现广域轨迹分析|
|**科研加速与学术基准**|动态SLAM研究者缺少统一的、标准化的、开源的对比基线，复现和评估算法费时费力|担当"裁判"角色，加速动态SLAM领域的科研迭代|

值得注意的是，在极端环境中，纯几何模型的鲁棒性优势尤为凸显。例如，**太空探索中**，卫星、岩石、碎片可能从未出现在任何训练数据中；**建筑工地、水下、灾区**中，重型机械、悬浮物、临时结构千奇百怪。在语义模型失效的"荒漠"中，纯几何方案几乎是唯一可靠的锚点。

### 🧩 结合YOLO-World/SAM填补的空白

如果说DynoSAM的几何核心如同"盲人的手杖"——通过感知运动来推断物体的存在和属性，那么YOLO-World与SAM就是为系统装上了一双能够**识别物体本质的"眼睛"**。

| 纯几何方法的天生盲区                                                                     | 语义结合的填补价值                                                                                   |
| ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------- |
| **盲区一：对静止可动物体"视而不见"** ——物体只要当前不动，其运动与背景完全一致，几何方法无法将其与场景结构区分。例如停在路边的车在下车后可能突然启动 | YOLO-World能**"认出它是车"**，并赋予"潜在可动"的先验属性；SAM提供精确像素级轮廓，系统可从第一帧起对其进行独立建模与跟踪，一旦该物体开始运动，瞬间锁定而非被动响应 |
| **盲区二：运动中断后的身份失忆** ——物体被遮挡后重新出现，或停下来再启动，几何关联链条断裂                               | 通过卷积外观特征实现**长期身份保持**：视觉特征为再出现的物体提供强识别的"指纹信息"，重建关联，告诉系统这是老朋友而非新面孔                            |
| **盲区三："泛化而不自知"——知道怎么动，不知是什么** ——能精确估算物体3D轨迹，却无法表述其语义类别（是人还是车）                  | 语义标签将轨迹抽象为时空事实：将SLAM地图转化为上层任务规划、人机交互、决策控制所需的结构化语义知识                                         |
| **盲区四：非刚体整体跟踪的歧义与不稳定** ——缺乏精确轮廓，仅靠稀疏特征点来表征整个物体，容易受特征漂移影响                       | SAM的像素级掩码提供精确的**轮廓约束**：优化时只使用掩码内高置信度的点，极大提升物体位姿估计的准确性，将相关物体位姿估计精度提高约60%（如OP-SLAM在相关数据集上所示）  |

### **初步拟定的实现架构**
具体实现上，可以采用三层递进的耦合架构：

| 耦合层              | 实现方式                                                                                  | 核心作用                    |
| ---------------- | ------------------------------------------------------------------------------------- | ----------------------- |
| **松耦合（数据预处理）**   | YOLO-World+SAM生成语义掩码后，作为独立节点发布"语义掩码"与"实例标签"消息，DynoSAM订阅接收并过滤动态点                       | 快速实现动静分离增强，帮助降噪         |
| **紧耦合（语义因子）**    | 将语义一致性建模为因子图中的一个新因子（衡量物体3D投影与SAM掩码边界的距离），与重投影因子、运动因子联合优化                              | 让语义信息反向约束和修正几何估计，实现双向促进 |
| **学习耦合（轻量外观模型）** | 采用之前讨论的卷积外观特征方案：①复用YOLO-World骨干网络输出的训练特征，提取RoI区域特征向量（零额外开销）；②维护基于历史帧平均的时序外观模型，提高匹配鲁棒性 | 增强长期数据关联，提升再识别能力        |

##### 1
是否在静态特征点足以支撑跟踪建图等活动的情况下，动态物体删得越多越好；
在高度动态场景下，如果动态物体被删除就会导致特征点不足，精度下降乃至于跟踪失败，这时才需要考虑利用动态特征点。

也许需要设立一个静态特征点阈值作为考虑。
